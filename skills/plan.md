# Skill - plan

## When To Use

- A spec, ADR, or analysis generates work.
- Decisions must become verifiable tasks.
- Roadmap, phase, workstream, task, step, and verification must be structured.

## Canonical Prompt

`prompts/05-create-task-plan-from-spec.md`

## Main Template

`templates/TASK_PLAN_TEMPLATE.md`

## Config

Read `pom.config.json` to respect test structure, source roots, official docs, and control severities before proposing verifications or paths.

## Output

- verifiable task plan;
- user scenarios;
- expected tests or lint;
- done criteria.
