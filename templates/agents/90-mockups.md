## Mockup Governance

When the project uses mockups, each mockup package must have a `MOCK_MANIFEST.md` that declares type (Complete/Partial), date, scope, impacted screens, processes, entities, and required decisions.

Use `pom/templates/MOCK_MANIFEST_TEMPLATE.md` for new manifests.

Reconciliation between mockups and wiki/docs/decisions must be documented. If a mockup contradicts the current wiki or an approved decision, surface the divergence and propose a reconciliation or new ADR.
